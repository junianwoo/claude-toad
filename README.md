# Claude Toad

**One command. Your entire Claude Code setup. Done.**

```
npx claude-toad init
```

You've written a CLAUDE.md, rewritten it, then read a blog post that told you everything you put in it was wrong. You've hit 300 lines before learning Anthropic recommends under 150, and that "as instruction count increases, instruction-following quality decreases uniformly." You've entered the settings.json maze where "a setting that works one day mysteriously breaks the next because it's being overridden by another file you forgot existed."

You've read 12 blog posts about configuring Claude Code. You're still not sure you did it right.

Claude Toad scans your actual project and generates a complete, project-specific `.claude/` directory in seconds.

Not a template. Not a starter kit. A configuration built from YOUR repo.

## Install

```
npx claude-toad init
```

No global install needed.

First run asks for your Anthropic API key. Stored locally at `~/.claude-toad/config.json`. Never sent anywhere except directly to Anthropic's API.

Already have `ANTHROPIC_API_KEY` set? Claude Toad finds it automatically.

## Four Commands

### `init` — Configure an existing project

Scans your repo. Calls Claude. Generates everything.

```
npx claude-toad init
```

### `new` — Start a project with Claude Code baked in

Interactive setup. Pick your stack, services, conventions. Get a project scaffold with a fully configured .claude/ directory from the first commit.

```
npx claude-toad new
```

### `package` — Share your config with your team

Bundles your .claude/ directory into an installable plugin. One person configures. Everyone benefits.

```
npx claude-toad package
npx claude-toad install my-team-config.zip
```

### `add-skill` — Generate custom skills from any source

Turn your own material into production-ready skills. Accepts YouTube videos, PDFs, documentation, URLs, audio files, and raw text. Integrates with [Smidge](https://smdg.app) for skill generation.

```
npx claude-toad add-skill --from "https://youtube.com/watch?v=..."
npx claude-toad add-skill --from "./docs/api-guide.pdf"
npx claude-toad add-skill --from "./onboarding/new-hire-training.mp4"
```

## What It Actually Does

**Step 1: Scans.** Reads your package.json, tsconfig, prisma schema, test configs, CI setup, .env references, README, and directory structure. No AI involved yet. Pure file system analysis. Fast.

**Step 2: Generates.** Sends the project fingerprint to Claude's API (your key, your tokens) across 4 focused calls on Opus. Each call has one job: CLAUDE.md, then skills, then agents, then commands. Focused calls produce better output than asking one call to generate 10 files. Settings and MCP config are built from templates, no API call needed.

**Step 3: Writes.** Drops everything into `.claude/`. Every file tagged `# Generated by Claude Toad — Review before use`. You're in control.

## What You Get

| File | What It Does |
|------|-------------|
| `CLAUDE.md` | Project context in under 150 lines. Stack, architecture, commands, conventions, gotchas. Claude reads this every session. |
| `skills/*.md` | Deep knowledge for your framework, database, and testing setup. Loaded only when relevant. Keeps CLAUDE.md lean. |
| `agents/*.md` | Code reviewer and PR writer pre-configured for your language and standards. |
| `commands/*.md` | Slash commands for test, deploy, and common workflows. Type `/test` instead of explaining your test setup every time. |
| `settings.json` | Permissions and tool configs. No `dangerously-skip-permissions`. Sensible defaults. |
| `.mcp.json` | MCP servers for your actual services. GitHub, database, whatever you use. Nothing you don't. |

## Why Not Just Use `/init`?

Claude Code's built-in `/init` generates a basic CLAUDE.md. That's it.

Claude Toad generates the entire `.claude/` directory: CLAUDE.md + skills + agents + commands + hooks + MCP config + settings. All project-specific. All following Anthropic's best practices for progressive disclosure, context budgeting, and the 150-line ceiling.

`/init` gives you a starting point. Claude Toad gives you the finish line.

---

## Why Not a Starter Kit?

Starter kits are static. They were written for someone else's project. You clone them, swap a few variables, and hope the 18 pre-built agents and 27 slash commands are relevant to what you're building.

Claude Toad generates configuration from your project. Not from a template. If you use Prisma, you get a Prisma skill. If you use Playwright, you get a Playwright skill. If you don't use either, you don't get files you'll never read cluttering your context window.

## Custom Skills

Claude Toad generates starter skills for your detected stack. For custom skills built from your own material, `add-skill` integrates with [Smidge](https://smdg.app) to turn source material into spec-compliant SKILL.md files.

```
npx claude-toad add-skill --from "https://youtube.com/watch?v=..."
npx claude-toad add-skill --from "./docs/internal-api-guide.pdf"
npx claude-toad add-skill --from "./training/onboarding-walkthrough.mp4"
```

Accepts YouTube videos, PDFs, docs, URLs, audio, and raw text. Smidge handles source extraction, frontmatter generation, progressive disclosure, and constraint levels so the output works across Claude Code, Cursor, Copilot, and 30+ agents.

Requires a [Smidge](https://smdg.app) account.
